# Contribution Guide

## Documentation Requirements

When changing data sources, API inputs, or outputs, update:

- About page
- README.md
- docs/CODEMAP.md
- relevant docs under /docs

When adding new insights, charts, or AI flows, update:

- README.md
- relevant docs under /docs

## Pull Request Requirements

Every PR must include explicit notes stating:

- whether this is a code file added or updated
- exact file paths affected
- exact changes made
- whether docs were updated
- whether data sources or credits were updated
- whether analysis behavior changed

Avoid vague notes such as "fixed issue" or "minor changes".

## Review Policy

- Always request review
- Do not self-merge
- Address review feedback explicitly

## Contributor Checklist

Before opening a PR, confirm:

- changes are scoped
- docs are updated
- data credits are complete
- PR notes are explicit
- review has been requested
